Gambaran Umum Sistem

Latar belakang, tujuan, dan stakeholder aplikasi monitoring kebun

🌾 Latar Belakang & Konteks Bisnis

Industri perkebunan tembakau memiliki siklus operasional yang kompleks, mulai dari persiapan lahan, penanaman, perawatan harian, panen, proses curing (aging), hingga kontrol kualitas produk jadi. Seluruh aktivitas ini sebelumnya dikelola secara manual menggunakan catatan kertas dan spreadsheet yang tidak terintegrasi, sehingga menghambat pengambilan keputusan berbasis data oleh manajemen.

Sistem Monitoring Operasional Kebun hadir sebagai solusi digital untuk mendigitalisasi seluruh alur kerja tersebut ke dalam satu platform terpadu yang dapat diakses secara real-time oleh semua pemangku kepentingan.

Flow Proses 1 – Autentikasi & Manajemen Akses

Alur login, verifikasi peran, dan kontrol akses berbasis role (RBAC)

📌 Deskripsi Proses

Sebelum dapat mengakses sistem, setiap pengguna wajib melalui proses autentikasi. Sistem menerapkan Role-Based Access Control (RBAC) dengan 3 tingkatan peran: Admin, Kepala Bagian (Kabag), dan Staf Lapangan. Setelah login berhasil, sistem akan mengarahkan pengguna ke tampilan yang sesuai dengan perannya.

🔄 Diagram Alur Autentikasi
1
Semua Pengguna
📱 Akses Halaman Login
Pengguna membuka aplikasi web di browser. Sistem mendeteksi tidak ada sesi aktif di localStorage, kemudian menampilkan form login.
React: Login.jsx localStorage check
2
Pengguna
🖊️ Input Kredensial
Pengguna memasukkan username dan password pada form login. Form mengirim data ke API endpoint auth.php via HTTP POST.
POST /auth.php JSON Body
3
Backend PHP + MySQL
🔍 Verifikasi Kredensial di Database
Server memverifikasi username dan password hash di tabel users database MySQL. Jika cocok, server mengembalikan objek user berisi: id, name, username, role.
MySQL Query password_verify()
4
Sistem (Frontend)
✅ Sesi Login Berhasil → Routing Berdasarkan Peran
Data user disimpan di localStorage sebagai sesi. Sistem kemudian menavigasi pengguna ke dashboard yang sesuai perannya secara otomatis.
Admin → Panel Admin Kabag → Dashboard Staf → Overview + Input
5
Sistem
❌ Login Gagal → Notifikasi Error
Jika kredensial salah, API mengembalikan error. Frontend menampilkan pesan kesalahan melalui sistem Toast Notification. Pengguna diminta mencoba ulang.
HTTP 401 Error Toast: "Login gagal"

📁 Alur Import Data via Excel
1
📤 Upload File Excel (.xlsx)
Staf mengupload file Excel berisi data pekerjaan. Sistem menggunakan library SheetJS (xlsx) untuk membaca file secara client-side tanpa upload ke server.
2

Sistem menampilkan preview baris data yang akan diimport. Pengguna dapat memverifikasi sebelum konfirmasi. Data yang tidak valid akan dilewati.

Sistem melakukan iterasi tiap baris dan menyimpan satu per satu ke MySQL via API. Toast notification menampilkan jumlah data berhasil diimport.
💡 Fitur Unggulan: Kalkulasi Capaian Otomatis
Saat staf mengisi form, sistem secara langsung (live preview) menghitung persentase capaian berdasarkan: Capaian (%) = (Realisasi S/D ÷ Rencana) × 100. Preview ini menampilkan status kinerja dengan warna: 🟢 ≥85% Baik, 🟡 50–84% Cukup, 🔴 <50% Perlu Perhatian.

Flow Proses 4 – Papan Alur Produksi (Kanban Board)

Manajemen status batch tembakau dari panen hingga produk jadi

📌 Deskripsi Proses

Kanban Board adalah visualisasi alur produksi tembakau yang memungkinkan staf lapangan untuk menggeser status batch secara drag-and-drop antar kolom proses. Setiap kolom mewakili satu tahapan dalam proses pengolahan tembakau pascapanen.

🔄 Alur Perubahan Status Batch
1
Sistem (Otomatis)
📋 Batch Baru Masuk Kolom "TO DO"
Setiap batch yang baru diinput akan otomatis masuk ke kolom TO DO sebagai status default. Kartu batch menampilkan: Batch ID, Uraian, Gramasi (Kg), Penanggung Jawab, Kebun, dan Tanggal.
2
Staf Lapangan
🖱️ Drag-and-Drop Kartu Batch ke Kolom Berikutnya
Staf menggeser kartu batch dari satu kolom ke kolom lain menggunakan fitur drag-and-drop HTML5. Selama penggeseran, kolom tujuan akan bersinar sebagai visual feedback.
HTML5 Drag API Hanya Staf yang bisa geser
3
Sistem
🔄 Validasi & Update Status di Database
Saat batch di-drop, sistem memverifikasi status baru berbeda dari status sebelumnya, kemudian mengirim request PUT ke API untuk memperbarui field status_proses di MySQL.
PUT /laporan.php?id=... status_proses field
4
Sistem
✅ Konfirmasi & Refresh Board
Setelah update berhasil, sistem menampilkan toast notifikasi "Batch X dipindahkan ke Y" dan me-refresh seluruh data board dari API untuk memastikan konsistensi data.
ℹ️ Pemetaan ke Proses Bisnis Nyata:
• TO DO = Lahan siap dipanen, belum mulai aktivitas
• IN PROGRESS = Proses pemetikan/panen daun tembakau sedang berlangsung
• AGING PROCESS = Daun memasuki tahap fermentasi & curing (pengovenan/pengeringan alami)
• TAHAP QC = Produk memasuki area inspeksi kualitas (gramasi, bentuk, wrapper)
• PRODUK JADI = Produk lulus QC dan siap masuk gudang / distribusi

Flow Proses 5 – Kontrol Mutu (QC) & Analisis Defect

Alur inspeksi kualitas produk dan manajemen reject

📌 Deskripsi Proses

📊 KPI & Analitik yang Dihasilkan Modul QC
Metrik QCFormulaInterpretasi
Total Diperiksa Lolos + Reject Volume inspeksi total
Total Lolos (Passed) Σ qcPassed Produk memenuhi standar
Total Reject Σ qcRejected Produk tidak memenuhi standar
Rasio Reject (%) (Reject ÷ Total) × 100 <10%: Normal, ≥10%: Peringatan
📈
Analisis Distribusi Cacat
  • Visualisasi bar chart per kategori cacat
  • Persentase tiap jenis cacat dari total reject
  • Identifikasi jenis cacat paling dominan
  • Dasar analisis root cause & perbaikan proses
  • Log tabel per batch dengan timestamp
🎯 Nilai Bisnis Modul QC: Data distribusi cacat membantu manajemen perkebunan mengidentifikasi bottleneck dalam proses produksi. Misalnya, jika Gramasi Tidak Sesuai mendominasi reject, maka perlu dilakukan kalibrasi ulang pada proses penimbangan atau penyesuaian SOP packaging.

🔌 Alur Data (Data Flow)
A
📱 Frontend (React) → 🖥️ Backend (PHP API)
Seluruh operasi data dilakukan melalui HTTP REST API calls dari frontend ke backend PHP. Metode yang digunakan: GET (baca data), POST (tambah baru), PUT (perbarui), DELETE (hapus).
GET /laporan.php POST /laporan.php PUT /laporan.php?id=X DELETE /laporan.php?id=X
B
🖥️ Backend (PHP API) → 🗄️ Database (MySQL)
PHP script memproses request, melakukan query SQL, dan mengembalikan response JSON ke frontend. Tabel utama: laporan_pekerjaan (data operasional) dan users (manajemen akun).
laporan_pekerjaan users JSON Response
C
📊 Integrasi Excel (SheetJS)
Untuk import/export Excel, sistem menggunakan library SheetJS yang berjalan sepenuhnya di browser (client-side). Tidak ada file Excel yang dikirim ke server — parsing dilakukan lokal untuk performa dan keamanan data.
SheetJS (xlsx) Client-side only
